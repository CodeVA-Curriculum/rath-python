Backstory

How do programs make decisions? Your music streaming service knows that if you press the pause button, then the music should pause. Programming if statements such as these are a foundational part of building programs that respond to user input or changing information. The if statement checks if a certain condition is True or False. Then, based on the True or False-ness of that condition, it runs different sections of code. This way the program can choose what to do next, if a certain condition is met.

If Statements

The following Python program shows what an ‚Äúif‚Äù statement looks like in action:

print("The beginning...")

if True:
    print("Option 1")

print("The End!")

The first line is just a print() statement like we‚Äôve been using for a while now; it‚Äôs nothing special. The last line is the same‚Äìjust a normal print() statement. The two lines in the middle are the important parts. First, you can see the if True: line; this is the beginning of our ‚Äúif‚Äù statement pattern. The if line is paired with a print() statement indented below it; this is the second part of our pattern. All ‚Äúif‚Äù statements will follow this pattern, or some variation of it.

The format of the ‚Äúif‚Äù statement is really important! The colon at the end of the if True: line needs to be there, and the line after the ‚Äúif‚Äù statement must be indented, otherwise the code will give you an error message when you try to run it. Lastly, you must capitalize the True or False value!

There are lots of variations on ‚Äúif‚Äù statements to try out. The one you experimented with above is the simplest version. Here‚Äôs how it works:

  1. The program runs normally until the computer reads the line that starts with if
  2. The program checks after the if to see if the value there is either True or False
  3. If the value is True, the program executes the line indented under the if line
  4. If the value is False, the program skips the line indented under the if line

There‚Äôs a slightly more complicated version to try out, the ‚Äúif-else‚Äù statement. In this version, the code doesn‚Äôt decide whether or not to skip a line; it chooses between two options based on the True or False value.

print("The beginning...")

if True:
    print("Option 1")
else:
    print("Option 2")

print("The End!")

Boolean Expressions

The True and False values are the most important parts of our ‚Äúif‚Äù statements; the code chooses different code sequences based on it. These values are Boolean data, named after George Boole. You may remember Boolean as one of the data types from the ‚Äúvariables‚Äù module. ‚ÄúIf‚Äù statements become very powerful when you start using them in conjunction with Boolean expressions; that‚Äôs what this section is all about.

Consider the following Python program:

print(5 > 2)

This program is written with a Boolean expression‚Äì5 > 2. Boolean expressions are sort of like mathematical expressions; the program will simplify the expression before printing it out. Boolean expressions are unlike mathematical expressions in that they don‚Äôt simplify down to a number; instead, the simplify down to a value of either True or False. The program above will print a value of True, because 5 is indeed greater than 2.

The > sign is called a Boolean operator. There are a few different kinds:

  • >: returns True when the value to the left is greater than the value to the right of the operator; otherwise returns False
  • <: returns True when the value to the left is lesser than the value to the right of the operator; otherwise returns False
  • ==: returns True when the values on either side of the operator are equal; otherwise returns False
  • >=: returns True when the value to the left is greater than or equal to the value to the right of the operator; otherwise returns False
  • <=: returns True when the value to the left is lesser than or equal to the value to the right of the operator; otherwise returns False

Note that the == operator is different than the = assignment operator that you used to create variables in the previous module. It‚Äôs a little bit confusing; the == operator is for Boolean expressions; the = operator is for assigning a value to a variable. Getting those two (admittedly similar) operators mixed up can become a source of very frustrating bugs in your code!

You can use Boolean operators to compare all different kinds of data, even strings (especially strings). Check out the code example below:

print("breakfast" == "lunch")

This program will output a value of False, because the string "breakfast" does not match the string "lunch". You can do the same thing with variables:

meal = "breakfast"
print(meal == "breakfast")

This time, the program will print True, because the variable called meal has the same value as the string "breakfast".

Interactive Stories

Let‚Äôs finally bring everything together! Consider the following code pattern:

print("You see a worn, wooden door in front of you. Do you OPEN it?")
answer = input()

if answer == "OPEN":
    print("You see a dark passageway before you...")
else:
    print("You stand alone in the darkness...")

print("The End!")

There‚Äôs one big difference between this ‚Äúif‚Äù statement and the ones you have seen thus far‚Äìinstead of using True or False, it uses a Boolean expression. When the variable called answer has the same value as "OPEN", the story continues. If not, the program prints "You stand alone in the darkness...". Spooky!

This story is pretty short, though. Here‚Äôs how we might continue it:

print("You see a worn, wooden door in front of you. Do you OPEN it?")
answer = input()

if answer == "OPEN":
    print("You see a dark passageway before you...")
    print("It's too dark to see. Do you want to turn on a FLASHLIGHT, or WALK?")
    answer = input()

    if answer == "FLASHLIGHT":
        print("The hallway has a large hole in the floor! You carefully step around it.")
    elif answer == "WALK":
        print("You take careful steps forward, and suddenly you fall through the floor!")
    else:
        print("That's not an option.")
        print("You stand alone in the darkness...")
else:
    print("You stand alone in the darkness...")

print("The End!")

We call this pattern a nested ‚Äúif‚Äù statement. You can add as much code as you want to the paths of an ‚Äúif‚Äù statement. Be careful though! As your story gets longer, it gets more and more difficult to read and add to. The cognitive load that you, as a programmer, have to deal with gets larger and larger as you add layers to your ‚Äúif‚Äù statements.

You also may have noticed a strange ‚Äúif‚Äù statement pattern in that previous example:

if answer == "FLASHLIGHT":
    print("The hallway has a large hole in the floor! You carefully step around it.")
elif answer == "WALK":
    print("You take careful steps forward, and suddenly you fall through the floor!")
else:
    print("That's not an option.")
    print("You stand alone in the darkness...")

This ‚Äúif‚Äù statement pattern has three options instead of just two. The line that says elif stands for ‚Äúelse if‚Äù, meaning that the code will check each Boolean expression in order. When the code finds a True expression, it runs the section of code indented beneath and skips the rest.

Your interactive narrative will mostly consistent of these ‚Äúif‚Äù statement patterns, so it‚Äôs a good idea to get familiar. Try out the experiment below to acquaint yourself!
